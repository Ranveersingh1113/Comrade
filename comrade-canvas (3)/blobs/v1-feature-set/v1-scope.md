# V1 scope

## V1 user

Any member of a small team of up to 4 people who notices communication lag or miscoordination and wants the AI to surface what's falling through the cracks — so they don't have to carry the coordination load alone.

## In v1

**Project memory**
- Bootstrapped at onboarding: team uploads docs, chat exports, links, or types a description
- Updated over time via GitHub sync and new manual uploads
- AI uses this as live context for everything it does

**In-platform communication**
- Team chat inside the platform room
- Primary communication channel during the pilot

**Deadline tracking**
- User-set deadlines
- AI is aware of them and references them when flagging gaps

**Contribution tracking**
- Score per member, visible to all
- Signals: GitHub commits, PRs, merges + platform activity (messages, docs, tasks)
- Task/role completion bar per member

**AI — four pillars**
- Monitor communication: reads all platform activity, detects who's active and who's gone quiet
- Surface accountability gaps: proactively flags missed deadlines, silent members, ownerless tasks
- Engage with documents: answers questions about uploaded docs and GitHub content, summarises project state on request
- Take action: creates and assigns tasks, sends private nudges to members, posts structured status updates — triggered by user instruction or automatically when a threshold is crossed

## Out of v1 (deferred)

- Live WhatsApp integration (replaced by export upload at onboarding)
- Auto-updating project context from platform activity alone (Option A)
- Deep GitHub analytics beyond contribution signals
- Contribution charts with historical graphs
- Mobile app (platform delivery decided in technical build phase)
